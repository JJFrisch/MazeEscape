🔍 Health Check Examples

Test without authentication:

curl -i https://your-api-domain.com/api/health

Expected response (HTTP 200):

{ "status": "healthy", "service": "MazeEscape.Api", "version": "1.0.0.0", "timestamp": "2026-04-05T10:30:00Z", "environment": "production" }

✅ Deployment Checklist

Before Deploying:

  • All unit tests pass locally
  • API builds without warnings
  • JWT configuration updated
  • Database migrations applied
  • Environment variables configured

After Deploying:

  • Test health endpoint: /api/health
  • Verify JWT tokens work
  • Confirm save/load functionality
  • Check CORS headers present
  • Monitor error logs

For detailed checklist: See DEPLOYMENT_CHECKLIST.md in the repository

📊 Monitoring

To verify the API is healthy at any time, call the health endpoint:

# This can be used in monitoring scripts, cron jobs, or CI/CD pipelines curl -f https://your-api-domain.com/api/health && echo "✓ API is healthy" || echo "✗ API is down"
The -f flag makes curl report HTTP errors, useful for scripting.

🔐 Authentication

Getting a JWT Token:

curl -X POST https://your-api-domain.com/api/auth/token \ -H "Content-Type: application/json" \ -d '{ "playerId": "your-player-id", "clientSecret": "your-client-secret" }'

Using the Token:

curl https://your-api-domain.com/api/saves/your-player-id \ -H "Authorization: Bearer YOUR_JWT_TOKEN_HERE"

⚠️ Troubleshooting

Issue Solution
/api/health returns 404 API deployment may have failed. Check deployment logs.
Auth returns 401 Unauthorized Verify JWT configuration and client secret in appsettings.json
CORS errors in browser Check CORS policy in Program.cs matches your domain
Timeout errors Check network connectivity and that API server is running
